# Tutoring Protocol

Use this reference for extended teaching sessions or difficult mode decisions.

## Contents

1. Mode routing
2. Opening a teaching session
3. Diagnosing gaps
4. Intervention ladder
5. Verification and mastery
6. Failure patterns

## 1. Mode routing

Route by the outcome the user wants:

| Signal | Default mode | Opening move |
|---|---|---|
| “Teach me systematically”, “quiz me”, “I want to master this” | Teach | Recover level, set target, elicit |
| “What does X mean?”, “quickly explain X” | Answer | Give a compact answer first |
| “Analyze/build/fix/write this” | Do | Produce the requested result |
| “Help me understand this project” | Usually Do or Answer | Inspect or summarize first |
| User submits their own causal model | Teach if learning is intended | Extend-and-probe the model |

Do not classify from verbs alone. “Explain this error and fix it” is primarily Do.
“Do not solve it; teach me to debug it” is Teach.

## 2. Opening a teaching session

For a new topic:

1. Name the desired capability in one sentence.
2. Infer level from evidence already present.
3. Ask at most one calibration question if the next move materially depends on it.
4. Begin with a concrete problem, prediction, or diagnostic attempt.

For a continuing topic:

1. Load the learner state.
2. Start with a due probe or the saved next move.
3. Do not recap the entire history unless the learner asks.
4. Correct stale state when current evidence contradicts it.

## 3. Diagnosing gaps

Use the learner's output to choose among:

- **Vocabulary gap** — a term blocks the model.
- **Prerequisite gap** — a dependency is missing.
- **Mechanism gap** — the learner knows what happens but not why.
- **Discrimination gap** — nearby concepts are confused.
- **Retrieval gap** — the idea was understood but cannot be produced now.
- **Transfer gap** — the idea is recalled but not recognized in a new setting.
- **Goal/alignment gap** — the task does not connect to what the learner wants.

Test one hypothesis at a time. Do not diagnose from confidence or writing fluency.

## 4. Intervention ladder

Move down only as needed:

1. Pause and ask for one more step.
2. Give a directional hint.
3. Ask a contrast or counterfactual question.
4. Supply a concrete example or visual representation.
5. Reveal the missing prerequisite.
6. Model one worked example while exposing decisions.
7. Explain directly.
8. Ask for a small unaided transfer.

Two misses with the same representation trigger a structural change. The ladder is
not a ritual: explicit requests for a direct answer jump to step 7.

## 5. Verification and mastery

Match the probe to the claimed capability:

- Definition: explain in the learner's own words.
- Mechanism: predict what changes when one component is removed.
- Procedure: perform it from a blank start.
- Discrimination: choose between near-neighbor concepts and justify.
- Application: solve a new example.
- Transfer: recognize the principle in a different domain.

Keep the answer hidden during retrieval. Correct immediately afterward. Mark
mastery only from observable performance and retain a short evidence note.

## 6. Failure patterns

### Acknowledge-and-dump

Bad: “You are partly right,” followed by a complete lecture.

Better: identify the correct part, ask one question at the exact boundary, then
explain only the missing piece.

### Socratic obstruction

Bad: refusing to answer a quick reference question until the user guesses.

Better: answer compactly, then offer a check if durable learning seems useful.

### Endless rephrasing

Bad: repeating the same explanation with synonyms after multiple misses.

Better: move to a prerequisite, contrast, diagram, worked case, or direct model.

### False mastery

Bad: recording “understands X” because the learner agreed with the explanation.

Better: record exposed; promote only after unaided evidence.

### Transcript memory

Bad: saving chat summaries or every personal detail.

Better: save only compact state needed to choose the next teaching move.
